Key insight

The walk-forward v2 strategy with risk overlay achieved the lowest drawdown of any version — 4.92% vs 35.75% for buy-and-hold. The negative total return reflects an overly conservative stance in a bull-dominated period (2010–2026), not model failure. The risk management architecture works; the parameter calibration needs tuning for return-seeking environments.

Transition persistence

The HMM transition matrix showed high diagonal values — each regime is highly self-persistent. This is the critical property that makes HMMs useful for trading: regimes don't flip every day. Walk-forward retraining every 3 months ensures the regime structure stays calibrated to current market conditions.

Technical design

Pipeline architecture

A production-grade ML pipeline with robust data ingestion, feature engineering, model training, and live prediction.

01
Data ingestion & cleaning
SPY, VIX, 10Y/5Y Treasury yields downloaded via yfinance. Handles MultiIndex columns, missing values, and inf replacement. Validates minimum data length before proceeding.
yfinance MultiIndex handling ^VIX, ^TNX, ^FVX
02
Feature engineering — 8 signals
Log returns, 20/60-day rolling volatility, intraday range, 10-day momentum, 50-day trend ratio, VIX level, 10Y–2Y yield spread, and 200-day momentum. All scaled with StandardScaler.
Log returns Vol_20 / Vol_60 VIX_Close Yield_Spread Momentum_200
03
GaussianHMM training
4-component Gaussian HMM with full covariance, 2,000 EM iterations. Dynamic regime labeling assigns Bull/Bear/Sideways/High-Volatility labels post-hoc based on each regime's learned mean return and volatility quantiles — no hardcoded thresholds.
hmmlearn covariance_type='full' predict_proba()
04
Walk-forward retraining
5-year rolling training window, retrained every 3 months. Each window fits a fresh scaler and HMM — preventing look-ahead bias. Results are stitched together across windows for realistic out-of-sample evaluation.
5-yr train window 3-month step No look-ahead bias
05
Risk overlay & volatility targeting
Position size is set by regime probability thresholds, then scaled by a volatility target (10% annualized, capped at 2× leverage). A 3-day minimum hold period prevents excessive turnover.
Soft classification Vol targeting 10% 5bps per trade 3-day min hold
06
Live regime prediction
predict_current_regime() takes any new DataFrame, validates features, scales, and returns the current regime ID, label, and full probability vector over all 4 states — suitable for live deployment.
predict_current_regime() Probability vector output Inference-ready
Strategy logic

Risk overlay rules

Instead of binary on/off signals, the strategy uses HMM output probabilities to size positions continuously — a soft classifier approach that avoids whipsawing.

Bull_Prob > 0.65
+1.0× to +1.2× leverage (vol-targeted)
0.40 < Bull_Prob ≤ 0.65
+0.6× leverage (vol-targeted)
Bull_Prob ≤ 0.40 AND Bear_Prob > 0.60
0.0× or −0.3× short (vol-targeted)
